01

클로드 3형제 이해하기

챗 · 데스크탑 · 코드, 뭐가 어떻게 다른가요?

지금까지 브라우저에서 claude.ai에 접속해 채팅했다면, 여러분은 "클로드 챗" 이용자예요. 같은 클로드지만 어디서, 어떤 권한으로 만나느냐에 따라 할 수 있는 일이 완전히 달라집니다. 먼저 세 가지를 구분해 볼게요.

구분클로드 챗 (웹)클로드 데스크탑클로드 코드
모습브라우저에서 채팅내 PC에 설치된 앱내 컴퓨터의 파일을 직접 다루는 AI 비서
할 수 있는 일질문·글쓰기·분석챗 + 데스크탑 전용 기능(MCP 연결, 클로드 코드 실행 등)파일 읽기·수정, 프로그램 만들기, 명령 실행, 자동화
파일 접근첨부한 파일만첨부 + 연결한 도구폴더 통째로 접근 가능
결과물채팅창 속 텍스트 (복사·저장은 내 몫)챗과 동일 + 아티팩트내 폴더에 진짜 파일로 저장됨
쓰는 곳어디서나 (모바일 포함)PC데스크탑 앱 안, 또는 터미널(명령창)
ℹ️
비유하자면

클로드 챗이 "전화 상담원"이라면, 클로드 코드는 "우리 집에 직접 와서 일해 주는 도우미"예요. 파일을 열어 보고, 고치고, 새로 만들고, 결과 확인까지 해 줍니다. 챗에서는 "이렇게 하시면 돼요"라는 설명을 들었다면, 코드에서는 "제가 다 해 놨어요"라는 결과를 받아요.

🛤️이 가이드의 두 가지 코스

클로드 코드는 같은 프로그램을 두 가지 방법으로 쓸 수 있어요. 이 가이드는 두 코스를 모두 다루되, 번호를 다르게 붙여 구분했습니다.

🟠

1부 — 데스크탑 코스 (D1~D3)

클로드 데스크탑 앱 안에서 버튼 클릭으로 클로드 코드를 켜는 방법. 별도 설치가 거의 없어 가장 쉬운 입문 코스예요. 오늘 처음이라면 여기부터!

🔵

2부 — 터미널 코스 (T1~T5)

검은 명령창에서 claude라고 입력해 실행하는 원조 방식. Git·Node.js 설치가 필요하지만 가장 강력하고 자유로워요. 1부에 익숙해진 뒤 도전하세요.

💡
어느 코스로 가야 할까요?

둘 중 하나만 골라야 하는 건 아니에요. 추천 경로: 1부(D1~D3)로 시작 → 감을 잡은 뒤 2부(T1~T5)로 확장. 두 방식은 계정·요금제를 공유하니 이중 결제 걱정도 없습니다. 3부(★)는 어느 쪽을 쓰든 공통으로 적용되는 활용편이에요.

02

준비물 체크리스트

시작 전에 이것만 확인하세요

  • 클로드 계정 — claude.ai에서 쓰던 계정 그대로 사용해요. 새로 만들 필요 없습니다.
  • 유료 요금제 (Pro 또는 Max) — 클로드 코드는 유료 구독에서 사용할 수 있어요. 무료 플랜에서는 동작하지 않습니다.
  • Windows 10 이상 PC — 이 가이드는 Windows 기준이에요. (Mac도 큰 흐름은 같습니다.)
  • Git (Windows PC라면 사실상 필수) — 데스크탑 Code 탭에서 로컬 폴더 작업을 하려면 꼭 필요하고, 터미널판에서도 강력 추천이에요. 설치 방법은 T2장에서 자세히 안내해요. 아직이라면 1부(D2) 시작 전에 미리 설치해 두는 걸 추천합니다.
  • 관리자 권한 — 프로그램 설치 시 "이 앱이 변경하도록 허용하시겠어요?" 창에서 "예"를 누를 수 있어야 해요.
  • 인터넷 연결 — 설치와 로그인에 필요합니다.
  • 여유 시간 — 1부는 20분, 2부까지는 커피 한 잔과 함께 1시간 정도 잡으세요.

💳요금제 간단 비교

플랜클로드 코드이런 분께
Free사용 불가웹 채팅만 가볍게 쓰는 분
Pro사용 가능 (사용량 보통)입문자에게 추천. 하루 몇 시간씩 쓰기에 충분
Max사용 가능 (사용량 넉넉)클로드 코드로 매일 장시간 작업하는 분
🔐
내 요금제 확인 방법

claude.ai 접속 → 왼쪽 아래 내 이름 클릭 → 설정(Settings) → 구독(Billing)에서 현재 플랜을 확인할 수 있어요. "Free"라면 Pro 이상으로 업그레이드해야 클로드 코드를 쓸 수 있습니다. 사용량은 일정 시간(약 5시간) 단위로 리셋되니, 한도에 닿아도 기다리면 다시 쓸 수 있어요. (참고: 학교·회사 단체 계정인 Team·Enterprise, 또는 개발자용 종량제 Console 계정으로도 쓸 수 있지만, 개인 이용자는 Pro·Max가 기본이에요.)

✅
내 PC가 Windows 몇인지 모르겠다면

윈도우 키 + R을 누르고 winver를 입력한 뒤 Enter — 버전이 적힌 창이 떠요. Windows 10 이상이면 통과입니다.

🟠

1부 — 데스크탑에서 쓰기

설치 하나로 끝나는 가장 쉬운 코스. 오늘 처음이라면 여기부터 시작하세요.

D1 ~ D3
D1

클로드 데스크탑 설치하기

첫 번째 설치 — 5분이면 끝나요

클로드 데스크탑은 웹의 claude.ai를 PC 앱으로 만든 것이에요. 웹과 똑같이 채팅할 수 있으면서, 웹에는 없는 기능 — 특히 클로드 코드 내장 실행 — 이 들어 있어요. 그래서 우리의 첫 설치 대상입니다.

📥다운로드부터 로그인까지

다운로드 페이지 접속

브라우저 주소창에 아래 주소를 입력해요. 클로드 공식 다운로드 페이지가 열립니다.

Windows용 설치 파일 받기

Download for Windows 버튼을 눌러요. Claude-Setup-x64.exe 같은 파일이 다운로드 폴더에 저장됩니다. (보통 수십 MB라 금방 받아져요.)

설치 파일 실행

내려받은 파일을 더블클릭해요. "이 앱이 디바이스를 변경하도록 허용하시겠어요?"가 뜨면 예를 누릅니다. 혹시 파란 "Windows의 PC 보호" 창이 뜨면 추가 정보 → 실행을 누르면 돼요. 설치는 자동으로 진행됩니다.

로그인

설치가 끝나면 클로드 앱이 자동으로 열려요. 기존 claude.ai 계정으로 로그인합니다. 구글 계정으로 가입했다면 "Continue with Google", 이메일 가입이라면 메일로 온 인증 코드를 입력하면 돼요.

설치 확인

웹에서 쓰던 것과 똑같은 채팅 화면이 뜨면 성공! 시작 메뉴와 작업 표시줄에 클로드 아이콘이 생겼는지도 확인해 보세요. 이제 이 앱이 여러분의 "클로드 본부"가 됩니다.

다운로드 주소 (브라우저 주소창에 붙여넣기)
https://claude.ai/download
⚠️
비슷한 이름에 주의

반드시 claude.ai 공식 페이지에서 받으세요. 검색 광고로 뜨는 가짜 사이트나 출처 불명의 설치 파일은 위험할 수 있어요. 주소창에 직접 입력하는 게 가장 안전합니다.

💡
웹이랑 뭐가 달라요?

채팅 자체는 웹과 같아요. 다른 점은 ① 클로드 코드를 앱 안에서 바로 실행, ② 커넥터(MCP)로 구글 드라이브·캘린더 같은 외부 도구 연결, ③ 단축키 Ctrl+Alt+Space로 어디서든 클로드 호출 — 이런 PC 전용 기능들이에요.

D2

데스크탑에서 클로드 코드 켜기

설치 없이 앱 안에서 바로 시작하는 가장 쉬운 방법

클로드 데스크탑 앱에는 클로드 코드가 이미 들어 있어요. 버튼 몇 번이면 바로 쓸 수 있습니다. 시작 전에 딱 하나, "작업 폴더" 개념만 이해하고 가요.

🔐
시작 전 필수 확인: Git 설치 여부

Windows에서는 Git이 설치돼 있어야 Code 탭에서 내 컴퓨터 폴더로 작업할 수 있어요(공식 요구사항이에요). 아직 Git을 설치하지 않았다면, 아래를 계속 읽기 전에 T2장으로 가서 5분 만에 설치부터 마치고 돌아오세요. (Mac은 대부분 Git이 기본 내장이라 이 단계가 필요 없어요.)

ℹ️
작업 폴더란?

클로드 코드에게 "이 폴더 안에서만 일해"라고 정해 주는 울타리예요. 클로드는 이 폴더 안의 파일을 읽고, 만들고, 고칠 수 있어요. 울타리 밖 파일은 기본적으로 건드리지 않으니, 폴더만 잘 정하면 안심하고 맡길 수 있습니다.

🚀첫 세션 시작하기

작업 폴더 미리 만들기 (추천)

파일 탐색기를 열어 바탕화면이나 문서 폴더에 클로드작업 같은 새 폴더를 하나 만들어 두세요. 우클릭 → 새로 만들기 → 폴더. 연습은 빈 폴더에서 시작하는 게 마음 편해요.

Code 탭 열기

클로드 데스크탑 상단 중앙의 Code 탭을 클릭해요. 유료 구독이 아니라면 업그레이드 안내가 뜨는데, 그 경우 02장에서 확인한 Pro/Max 구독이 되어 있는지 다시 확인하세요.

환경은 Local, 폴더는 방금 만든 곳으로

실행 환경을 고르는 화면이 나오면 반드시 Local(로컬)을 선택하세요 (Remote·SSH는 클라우드·원격 서버용이라 지금은 필요 없어요). 이어서 Select folder를 눌러 아까 만든 클로드작업 폴더를 지정해요.

첫 요청 입력

채팅창처럼 생긴 입력칸에 하고 싶은 일을 평소 말하듯 한국어로 적어요. 예: "이 폴더에 자기소개 웹페이지를 만들어 줘".

변경 사항 검토하고 승인

기본값인 Manual 모드에서는 클로드가 파일을 고치기 전에 무엇이 바뀌는지 미리보기(diff)로 먼저 보여줘요. 내용을 확인하고 Accept(수락)를 누르면 실제로 반영되고, Reject(거절)를 누르면 취소돼요. 이 확인 절차가 여러분의 안전장치예요.

결과 확인

작업이 끝나면 폴더에 실제 파일이 생겨 있어요. 파일 탐색기로 열어 확인해 보세요. html 파일이라면 더블클릭으로 브라우저에서 바로 열려요.

💡
첫 연습으로 좋은 요청

"이 폴더에 hello.html이라는 파일을 만들고, 열면 '안녕하세요'가 크게 보이게 해 줘" — 파일이 진짜로 생기는 걸 눈으로 확인하면 클로드 코드가 뭔지 단번에 감이 와요. 만든 파일이 마음에 안 들면 "다 지워 줘"라고 하면 됩니다.

⚠️
이런 폴더는 지정하지 마세요

C 드라이브 전체, "문서" 폴더 전체, 회사·학교의 중요 자료가 든 폴더처럼 범위가 너무 넓거나 소중한 원본이 있는 곳은 피하세요. 연습은 새 폴더에서, 실전은 필요한 파일만 복사해 온 폴더에서 하는 습관이 안전합니다.

기본 사용법과 활용 예시

요청하는 요령만 알면 실력이 확 늘어요

💬요청은 이렇게 하세요

  • 목표를 말하세요, 방법 말고 — "for문 짜 줘"보다 "학생 명단에서 미제출자만 골라내 줘"가 훨씬 잘 통해요. 방법은 클로드가 알아서 찾습니다.
  • 배경을 한 줄 곁들이기 — "나는 초등학교 교사고, 학부모에게 보낼 안내문이야"처럼 맥락을 주면 결과물의 말투·수준이 달라져요.
  • 한 번에 한 가지씩 — 큰 일은 잘게 나눠 요청하면 중간중간 확인하며 갈 수 있어요.
  • 마음에 안 들면 바로 말하기 — "버튼을 더 크게", "색을 파랑으로", "말투를 부드럽게" 같은 후속 요청으로 다듬어요. 몇 번이고 고쳐 달라고 해도 됩니다.
  • 모르면 물어보기 — "지금 뭘 한 거야?", "이 파일은 왜 만들었어?"라고 물으면 친절하게 설명해 줘요.

터미널 첫걸음

검은 창이 무섭지 않게 되는 10분 오리엔테이션

터미널(명령창)은 마우스 대신 글자로 컴퓨터에게 일을 시키는 창구예요. 영화 속 해커의 검은 화면이 바로 이건데, 실체는 아주 단순합니다. 명령을 한 줄 입력하고 → Enter를 누르면 → 컴퓨터가 실행하고 결과를 보여 준다 — 이게 전부예요.

ℹ️
왜 굳이 터미널인가요?

클로드 코드는 원래 터미널 프로그램으로 태어났어요. 터미널판은 ① 어떤 폴더에서든 즉시 실행, ② 세부 설정·확장 기능(MCP, 훅, 서브에이전트 등) 전부 사용 가능, ③ 다른 개발 도구와 자유롭게 조합 — 이 세 가지가 강점이에요. 그리고 걱정과 달리, 터미널을 열고 claude를 실행한 뒤에는 다시 한국어 채팅이라 어렵지 않습니다.

🖥️PowerShell 열기 — 3가지 방법

Windows의 기본 터미널은 PowerShell(파워셸)이에요. 아무 방법이나 편한 걸로 여세요.

  1. 윈도우 키를 누르고 powershell 이라고 검색 → Enter (가장 기본)
  2. 윈도우 키 + X → 메뉴에서 터미널 클릭 (Windows 11)
  3. 파일 탐색기에서 원하는 폴더를 연 뒤, 주소창에 powershell 입력 → Enter (그 폴더에서 바로 열려서 편해요!)

📖화면 읽는 법

터미널을 열면 이런 글자가 보여요:

터미널 화면의 의미
PS C:\Users\내이름>  ←  "지금 C드라이브의 Users\내이름 폴더에 있어요.
                        명령을 입력하세요"라는 뜻
  • PS — PowerShell이라는 표시예요.
  • C:\Users\내이름 — 지금 내가 "서 있는" 폴더의 주소(경로)예요. 터미널은 항상 어떤 폴더 안에서 일해요.
  • > — "입력 대기 중"이라는 커서. 여기에 명령을 칩니다.

🎮딱 3개만 연습해 봐요

지금 폴더의 내용 보기 — dir

dir 입력 → Enter. 지금 폴더 안의 파일·폴더 목록이 주르륵 나와요. 파일 탐색기로 폴더를 여는 것과 같은 일이에요.

다른 폴더로 이동 — cd

cd Desktop 입력 → Enter. 바탕화면 폴더로 이동해요. cd .. 은 "한 단계 위로". 폴더 이름을 몇 글자만 치고 Tab을 누르면 자동완성되니 긴 이름도 걱정 없어요.

화면 정리 — cls

화면이 지저분해지면 cls → Enter. 화면만 깨끗해지고 아무것도 삭제되지 않아요.

💡
복사·붙여넣기

터미널에서도 Ctrl+C(복사), Ctrl+V(붙여넣기)가 그대로 돼요. 이 가이드의 검은 상자에서 "복사" 버튼을 누르고 터미널에 Ctrl+V → Enter 하면 됩니다.

✅
터미널이 망가뜨릴까 봐 무섭다면

안심하세요. dir, cd, cls 는 보기만 하는 명령이라 아무것도 바꾸지 않아요. 이 가이드에 나오는 명령만 따라 하면 위험한 일은 없습니다. 모르는 명령을 인터넷에서 복사해 실행하는 것만 조심하세요.

☁️GitHub까지 가고 싶다면 (선택)

GitHub(깃허브)는 내 컴퓨터의 Git 기록을 인터넷에 올려 백업·공유·무료 웹사이트 호스팅까지 해 주는 서비스예요. github.com에서 무료 계정을 만든 뒤, 전용 도구 gh를 설치하면 클로드가 업로드까지 대신해 줄 수 있어요.

gh auth login                 # 브라우저로 GitHub 로그인 (화살표로 GitHub.com → HTTPS → 브라우저 선택)

설치 후 클로드에게 "이 프로젝트를 GitHub에 올려 줘"라고 하면 저장소 생성부터 업로드까지 알아서 진행해 줘요. 만든 웹페이지를 인터넷에 공개하고 싶다면 "GitHub Pages로 배포해 줘"라고 이어서 부탁하면 됩니다.

🔐
개인정보 주의

GitHub에 올린 파일은 (공개 저장소라면) 전 세계에 공개돼요. 학생 명단, 연락처, 비밀번호가 든 파일은 절대 올리면 안 됩니다. 올리기 전에 클로드에게 "개인정보가 들어 있는 파일이 있는지 확인해 줘"라고 검사를 시키는 것도 좋은 습관이에요.

🔧자주 겪는 문제

  • "claude는(git은/npm은) 인식할 수 없습니다" — 십중팔구 설치 후 터미널을 새로 안 열어서예요. 창을 모두 닫고 새 PowerShell을 여세요. 그래도 안 되면 PC를 재부팅해 보세요. 그래도 안 되면 PATH 문제이니 재설치가 빠릅니다 (Git 설치 시 PATH 화면에서 가운데 옵션 선택 확인).
  • 로그인 브라우저가 안 열려요 — 터미널에 표시된 주소(URL)를 복사해 브라우저에 직접 붙여넣으면 같은 로그인 화면이 열려요.
  • "플랜이 필요합니다"라고 나와요 — 클로드 코드는 Pro/Max 유료 구독이 필요해요. claude.ai 설정 → 구독에서 현재 플랜을 확인하세요.
  • 사용량 한도에 걸렸어요 — 구독 플랜은 일정 시간마다 사용량이 리셋돼요. 잠시 후 다시 시도하거나, 긴 대화는 /compact로 압축하고 새 주제는 /clear로 시작하면 사용량을 아낄 수 있어요.
  • 한글·공백 폴더 이름 때문에 이동이 안 돼요 — 폴더 이름에 공백이 있으면 따옴표로 감싸세요: cd "내 문서\클로드 작업". 자동완성(Tab)을 쓰면 실수가 없어요.
  • 클로드가 엉뚱한 파일을 고치려 해요 — 당황하지 말고 거부 버튼(또는 Esc)을 누른 뒤, "그 파일 말고 ○○ 파일을 고쳐 줘"라고 다시 알려주세요. @파일명으로 지목하면 더 확실해요.
  • 클로드가 만든 결과물이 실행이 안 돼요 — 오류 메시지를 그대로 복사해서 "이 오류가 났어. 고쳐 줘"라고 붙여넣으세요. 오류 해결이야말로 클로드가 가장 잘하는 일이에요.
  • 회사·학교 네트워크에서 설치가 안 돼요 — 보안 정책으로 다운로드가 차단된 경우예요. 네트워크 관리자에게 문의하거나 다른 네트워크에서 설치해 보세요.
  • "설치 완료"라고 나왔는데 claude 명령이 안 먹혀요 — 드물게 PowerShell 설치 스크립트가 성공 메시지만 띄우고 실제 파일은 못 받는 경우가 있어요(알려진 이슈). claude --version으로 먼저 확인하고, 안 되면 T4장의 CMD 대체 명령이나 winget install Anthropic.ClaudeCode로 다시 설치해 보세요.
  • 업데이트는 어떻게 하나요? — 터미널에서 claude update 한 줄이면 최신 버전으로 올라가요(방법 A로 설치했다면 평소엔 자동 업데이트돼요). 데스크탑 앱도 자동 업데이트됩니다. winget으로 설치했다면 winget upgrade Anthropic.ClaudeCode로 직접 갱신해야 해요. 설치 상태가 의심되면 claude doctor로 진단하세요.
